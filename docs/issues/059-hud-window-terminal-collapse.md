# #59 — HUD 윈도우 터미널 반응형 축소 설계 개선

**분류**: design
**심각도**: medium
**등록일**: 2026-03-13
**상태**: open

## 현상

Windows Terminal에서 터미널 폭이 좁아질 때 HUD가 **한 줄로 과도하게 축소**되어 정보 가독성이 크게 저하됨.

- 폭이 줄어들면 점진적으로 정보를 생략하는 것이 아니라, 갑자기 한 줄로 압축됨
- 사용자가 분할 화면이나 좁은 터미널을 쓸 때 매번 불편함 발생
- 현재 breakpoint 기반 tier 설계가 Windows Terminal 환경에 적절하지 않음

## 기대 동작

1. **점진적 축소**: 폭이 줄어들면 단계별로 정보 요소를 하나씩 제거 (갑자기 한 줄로 점프하지 않음)
2. **최소 2줄 보장**: 극소 폭(< 40col)이 아닌 이상 최소 2줄 유지
3. **우선순위 기반 정보 표시**: 폭이 좁을수록 중요 정보(모드, 상태)만 남기고 부가 정보(시간, 토큰) 순서로 제거

- Windows Terminal 기본 폭: 120col
- 분할 시 60col, 3분할 시 40col 수준
- ANSI 214 (Amber) 브랜드 컬러 유지 필수

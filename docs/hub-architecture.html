Architecture Comparison

tfx-auto Without Hub

기존 CLI 실행 방식 (Sequential & File-based)

  • 중앙 집중형 제어: 모든 결정이 오케스트레이터에 의존
  • 에이전트 간 통신 불가능 (Isolated execution)
  • 컨텍스트 전달: 오직 파일 시스템 공유를 통해서만 가능
  • 결과 수집: stdout/stderr 로그 파싱 (후속 작업 블로킹)
  • 상태 판단: 오직 프로세스 종료 코드로만 수행

tfx-hub With Hub

메시지 버스 기반 실시간 통신 (Real-time & Pub/Sub)

  • 실시간 메시지 교환 (ask/publish/handoff)
  • 토픽 기반 Pub/Sub 팬아웃으로 대규모 정보 동기화
  • 컨텍스트 전달: 브릿지 폴링을 통한 자동 주입
  • Lease 기반 상태 관리 (Heartbeat)
  • HITL: CAPTCHA나 승인 요청을 비동기적으로 처리

server.mjs

Streamable HTTP MCP 서버 진입점입니다. Express 의존성 없이 Node.js http 모듈만으로 경량 구현되어 있습니다.

  • /mcp: SSE 기반 MCP 전송 레이어
  • /bridge: CLI를 위한 REST API 엔드포인트

  • CORS 지원으로 로컬 도구 간 호환성 확보

v3.1.0-dev.4 Highlights

Pooled Random (UUIDv7) crypto.randomBytes(256) 풀을 사용하여 UUIDv7 생성 성능 극대화 및 시스템 entropy 고갈 방지.
EventEmitter Router ask() 요청에 대해 SQLite 폴링 대신 Node.js 내부 이벤트를 통한 즉시 응답(Low Latency) 구현.
Session Heartbeat 30분(TTL) 비활성 세션 자동 해제 및 Lease 기반 에이전트 생명주기 관리 안정화.
Lite REST Bridge 쉘 스크립트(tfx-route.sh) 호환성을 위한 원격 호출용 /bridge 엔드포인트 세트 최적화.

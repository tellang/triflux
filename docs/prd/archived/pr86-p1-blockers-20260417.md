- prompt: |
    PR #86 feat/codex-mcp-progress 브랜치 위에서 아래 P1 블로커 3건을 수정한다.

    **Fix 1 — `_inflightRejectors` leak**
    파일: hub/workers/codex-app-server-worker.mjs, line 801-850 부근 execute() 본문
    증상: execute()의 세 early-return 경로(thread/start 실패, thread id missing, turn/start 실패)가 this._inflightRejectors.delete(inflightReject)를 호출하지 않아 stale rejector가 영구 누적됨. PoC로 6회 연속 실패 시 Set size 6 확증됨.
    조치: execute() 본문을 try { ... } finally { offAll(); this._inflightRejectors.delete(inflightReject); } 로 감싼다. 기존 성공 경로와 timeout 경로의 delete/offAll 호출은 finally로 통합하고 인라인 호출은 제거한다.

    **Fix 2 — Stale timeout timer SIGTERMs reused child**
    파일: hub/workers/codex-app-server-worker.mjs, line 858-879
    증상: execute()의 timeout Promise가 setTimeout으로 타이머를 만들지만 resultPromise가 Race에서 이기면 clearTimeout을 호출하지 않음. 타이머가 unref만 되어 stale 상태로 남고, 나중에 fire되면 this.child.kill("SIGTERM") 실행. worker 재사용 시 이전 execute의 stale timer가 현재 execute의 child를 kill하는 race condition.
    조치: (1) timeout 콜백 내부에서 this.child 대신 closure로 capturedChild를 참조한다(외부 const capturedChild = this.child). (2) const timer 핸들을 보관하고 Fix 1의 finally에서 clearTimeout(timer)를 호출한다. (3) timer.unref 호출은 유지한다.

    **Fix 3 — stdin/stdout unhandled stream error crashes hub**
    파일: hub/workers/lib/jsonrpc-stdio.mjs, line 116-121 (생성자) + line 287-289 (_writeFrame)
    증상: 생성자에서 this._stdin, this._stdout의 error 이벤트를 구독하지 않고, _writeFrame(frame)은 stdout.write(line) 리턴값과 콜백을 무시. child.stdin EPIPE/ERR_STREAM_DESTROYED 발생 시 Node가 unhandled 'error' emit → uncaughtException → hub 프로세스 crash.
    조치:
    - 생성자에서 this._stdin.on("error", (err) => this._handleStreamError("stdin", err)) 와 this._stdout.on("error", ...) 추가
    - 새 메서드 _handleStreamError(which, err): JsonRpcTransportError로 래핑 후 this._emitError 호출, 이어 this._closeWith("closed", wrappedErr)
    - _writeFrame: this._stdout.write(line, (err) => { if (err) this._handleStreamError("stdout-write", err); })로 콜백 추가. state가 closing/closed일 때는 write 시도 건너뛰기(조기 return).

    **회귀 테스트 추가 (3건)**

    tests/unit/codex-app-server-worker.test.mjs:
      it("inflightRejectors cleanup on execute failures"): fake transport로 thread/start reject 시키고 execute()가 rejects 되더라도 worker._inflightRejectors.size === 0 인지 확인. 5~6회 반복.
      it("stale timeout timer does not kill reused child"): 짧은 timeoutMs로 A 실행해 완료되면, 이후 B를 execute(긴 timeoutMs)하는 동안 A의 stale timer가 fire되더라도 child가 kill되지 않는지 확인.

    tests/unit/jsonrpc-stdio.test.mjs:
      it("stdin 'error' event rejects pending requests with JsonRpcTransportError"): stdin.emit('error', new Error('EPIPE')) 후 진행 중 request가 JsonRpcTransportError로 reject 되는지 확인. 프로세스는 crash하지 않아야 함.

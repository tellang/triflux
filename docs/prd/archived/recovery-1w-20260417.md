# PRD: 1주일 유실된 개발 맥락 복구 (2026-04-10 ~ 2026-04-17)

## 목표
지난 1주일간 여러 세션/스웜/PR/이슈/메모 전반에서 축적됐지만 현재 main 또는 닫힌 추적 상태로 반영되지 않은 **유의미한 개발 맥락**을 4개 영역 병렬로 마이닝하여, 각 영역별 리포트 + 종합 로드맵을 생성한다. 읽기 전용 조사 — **코드/설정 변경 금지**. 출력은 전부 `docs/recovery/*.md`.

## 필수 분류 카테고리 (모든 shard 공통 — 각 발견 항목에 `type:` 반드시 명시)

| type | 의미 | 예시 |
|---|---|---|
| **context** | 배경/맥락 기록 (단순 정보) | "이 세션에서 X 설계 논의함" |
| **diff** | 같은 파일/주제에 여러 버전 수정이 나뉘어 존재 | "A가 X 수정, 나중에 B가 같은 줄 수정" |
| **reason** | 왜 그 수정/결정이 됐는지 근거 | "이 설정 끈 이유 = Y 에러 피하려고" |
| **intent** | 원래 의도했으나 결과와 다름 | "OMC 의존 없앨 의도였으나 `oh-my-claudecode:*` 그대로 씀" |
| **omission** | 구현/기록됐어야 하는데 누락 | "issue close됐지만 코드 커밋 없음" |
| **bug** | 실제 버그/에러/오작동 | "refresh_token_reused 에러 재발" |
| **loss** | 작업이 사라짐/추적 불가 | "stash 내용이 이제 적용 불가한 base" |
| **overwrite** | 한 수정이 다른 수정으로 덮여짐 | "55b2086이 PR #82 머지로 main에서 유실" |

각 리포트의 발견 항목은 **반드시** 위 type 중 하나(또는 여러 개) 라벨을 붙인다. 미분류 항목은 거부.

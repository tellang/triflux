## 2. 아키텍처 변경

### Before (v5.x)
```
/tfx-auto "A + B + C" → triage → Agent(slim wrapper) × N → Bash(tfx-route.sh)
                                  ↑ 래퍼당 ~200 토큰, Bash blocking, 실시간 불가
```

### After (v6.0.x)
```
/tfx-auto "A + B + C" → triage → headless.runHeadlessInteractive()
                                  ↑ 래퍼 0개, 실시간, WT 스플릿, 토큰 절감
```

### 핵심 API

```javascript
// hub/team/headless.mjs
runHeadless(sessionName, assignments, opts)         // 기본 실행 + cleanup
runHeadlessWithCleanup(assignments, opts)            // auto cleanup
runHeadlessInteractive(sessionName, assignments, opts) // 세션 유지 handle

// handle: { results, dispatch(), capture(), snapshots(), waitFor(), kill() }

autoAttachTerminal(sessionName, opts, workerCount)   // WT 스플릿
applyTrifluxTheme(sessionName)                       // psmux 테마
ensureWtProfile(workerCount)                         // WT 프로필 생성
```

### 실행 흐름
```
1. createPsmuxSession (lead pane 1개)
2. applyTrifluxTheme (status bar + border)
3. autoAttachTerminal (WT 가로 스플릿, 포커스 복원)
4. progressive split-window (워커 하나씩 추가)
5. dispatchCommand (CLI 실행)
6. waitForCompletion (onPoll + progressIntervalSec)
7. 결과 수집 + handle 반환
8. 후속 dispatch 가능 (세션 유지)
9. kill() → 정리
```
